# Context: Xây dựng Base Component System — UntitledUI Style

> **Mục đích file này:** Cung cấp đủ ngữ cảnh để AI tái tạo chính xác hệ thống base components theo đúng pattern, cấu trúc, và naming convention của repo `untitleduico/untitledui-nextjs-starter-kit`. Mọi quyết định kỹ thuật trong file này đều rút ra trực tiếp từ source code thực tế của repo đó.

**⚠️ Điểm khác biệt quan trọng so với Tailwind v3:**
- Không có file `tailwind.config.ts` — mọi token khai báo trong `src/styles/theme.css` bằng cú pháp `@theme { }`.
- Không dùng `@apply` cho base styles — dùng Tailwind utility class trực tiếp trong JSX.
- Cú pháp plugin: `@plugin "@tailwindcss/typography"` (trong `.css`), không phải `plugins: [require(...)]`.

### `src/providers/router-provider.tsx`

```tsx
"use client";

import type { PropsWithChildren } from "react";
import { useRouter } from "next/navigation";
import { RouterProvider } from "react-aria-components";

declare module "react-aria-components" {
    interface RouterConfig {
        routerOptions: NonNullable<Parameters<ReturnType<typeof useRouter>["push"]>[1]>;
    }
}

export const RouteProvider = ({ children }: PropsWithChildren) => {
    const router = useRouter();
    return <RouterProvider navigate={router.push}>{children}</RouterProvider>;
};
```

```tsx
"use client";

import { ThemeProvider } from "next-themes";

    );
}
```

Toàn bộ token khai báo trong `@theme { }`. Đây là cú pháp Tailwind v4 — không dùng `tailwind.config.ts`.

## 5. Base Components — Pattern chung

### 5.1 Bộ công cụ styling

Mọi component đều dùng 2 utility từ `@/utils/cx`:

```ts
// cx = tailwind-merge mở rộng để hiểu display-xs, display-sm...
cx("base-class", condition && "conditional-class", className)

// sortCx = giữ nguyên object, chỉ để Prettier/IDE sort được className
// Dùng để nhóm styles theo logic (common, sizes, colors)
sortCx({
    common: { root: "...", icon: "..." },
    sizes: { sm: { root: "..." }, md: { root: "..." } },
    colors: { primary: { root: "..." }, secondary: { root: "..." } },
})
```

### 5.2 "use client" directive

Tất cả component trong `base/` đều có `"use client"` ở đầu file vì dùng state, event handler, hoặc react-aria.

### 5.3 React Aria Components làm primitive

Component KHÔNG wrap `<button>`, `<input>` HTML thuần. Luôn dùng:

| Primitive | React Aria wrapper |
|---|---|
| Button | `Button as AriaButton` từ `react-aria-components` |
| Link | `Link as AriaLink` từ `react-aria-components` |
| Input | `Input as AriaInput`, `TextField as AriaTextField`, `Group as AriaGroup` |
| Select | `Select`, `ComboBox`, `ListBox` từ `react-aria-components` |
| Tooltip | custom `Tooltip` + `TooltipTrigger` trong `base/tooltip/tooltip.tsx` |

Lý do: React Aria cung cấp accessibility (keyboard nav, ARIA attributes, focus management) tự động.

### 5.4 Dual Button/Link pattern

`Button` và `ButtonUtility` đều hỗ trợ render thành `<button>` hoặc `<a>` tùy có `href` hay không:

```tsx
const href = "href" in otherProps ? otherProps.href : undefined;
const Component = href ? AriaLink : AriaButton;
```

### 5.5 Icon pattern

Icon nhận vào dạng FC (function component) hoặc ReactNode, phân biệt qua `isReactComponent()`:

```tsx
// src/utils/is-react-component.ts
import { isValidElement } from "react";
export const isReactComponent = (value: unknown): value is React.FC =>
    typeof value === "function" && !isValidElement(value);

// Dùng trong component:
{isValidElement(IconLeading) && IconLeading}
{isReactComponent(IconLeading) && <IconLeading data-icon="leading" className={styles.common.icon} />}
```

Icon từ `@untitledui/icons` import theo tên:
```ts
import { User01, Eye, EyeOff, HelpCircle, InfoCircle, X } from "@untitledui/icons";
```

## 7. Component API Reference

### 7.1 `Button` (`buttons/button.tsx`)

```tsx
// Sizes: "xs" | "sm" | "md" | "lg" | "xl"  (default: "sm")
// Colors:
//   "primary"              → bg-brand-solid, white text, skeuomorphic shadow
//   "secondary"            → bg-primary, ring-1, secondary text
//   "tertiary"             → transparent bg, hover bg
//   "link-color"           → no padding, brand text, underline on hover
//   "link-gray"            → no padding, tertiary text, underline on hover
//   "primary-destructive"  → bg-error-solid, white text
//   "secondary-destructive"→ bg-primary, error ring, error text
//   "tertiary-destructive" → transparent, error text
//   "link-destructive"     → no padding, error text

<Button size="md" color="primary" iconLeading={PlusIcon} isLoading={false}>
    Tạo mới
</Button>

// Dùng như link:
<Button href="/explore" color="secondary">Khám phá</Button>
```

**Pattern quan trọng:**
- `isDisabled` thay vì `disabled` (từ React Aria)
- `isLoading` thay vì `loading` — tự render spinner, ẩn children
- `iconLeading` / `iconTrailing` nhận FC hoặc ReactNode
- Khi icon-only (không có children): tự nhận `data-icon-only` → padding thay đổi
- Inner gradient border: `before:absolute before:inset-px before:border before:border-white/12 before:mask-b-from-0%` (chỉ primary/destructive)

### 7.2 `ButtonUtility` (`buttons/button-utility.tsx`)

```tsx
// size: "xs" | "sm"  (default: "sm")
// color: "secondary" | "tertiary"  (default: "secondary")

<ButtonUtility icon={Settings01} tooltip="Cài đặt" tooltipPlacement="top" size="sm" />
```

Icon-only button, tự wrap `<Tooltip>` nếu có `tooltip` prop.

### 7.4 `Avatar` (`avatar/avatar.tsx`)


### 7.5 `Input` (`input/input.tsx`)


## 8. Naming conventions

| Loại | Pattern | Ví dụ |
|---|---|---|
| File | kebab-case | `button-utility.tsx`, `avatar-label-group.tsx` |
| Component export | PascalCase | `ButtonUtility`, `AvatarLabelGroup` |
| Style object biến | `styles` (const) | `export const styles = sortCx({...})` |
| Prop boolean | `is` prefix từ React Aria | `isDisabled`, `isLoading`, `isInvalid`, `isRequired` |
| Event handler prop | `on` prefix | `onButtonClick`, `onChange` |
| Icon prop | `icon`, `iconLeading`, `iconTrailing` | FC type |
| Size variant | `"xs" \| "sm" \| "md" \| "lg" \| "xl"` | string literal |
| Color/tone variant | string literal | `"primary" \| "secondary" \| "brand" \| "error"` |
| Token class trong JSX | semantic alias | `text-primary`, `bg-secondary`, `ring-brand` |

## 9. Tailwind class conventions trong JSX

```tsx
// ✅ Dùng semantic token
className="text-primary bg-secondary ring-1 ring-primary"

// ✅ Dùng brand token
className="bg-brand-solid text-white hover:bg-brand-solid_hover"

// ✅ Utility color cho badge/icon (thông qua token)
className="bg-utility-brand-50 text-utility-brand-700 ring-utility-brand-200"

// ✅ Custom text size
className="text-display-sm font-semibold"

// ✅ Focus ring
className="outline-brand focus-visible:outline-2 focus-visible:outline-offset-2"
// hoặc:
className="outline-focus-ring focus-visible:outline-2"

// ✅ Dark mode (qua ThemeProvider — KHÔNG dùng "dark:" prefix Tailwind)
// Dark mode tự động qua @custom-variant dark trong globals.css
// Vì ThemeProvider thêm class ".dark-mode" vào <html>

// ✅ Conditional với cx:
className={cx("base", isActive && "font-semibold", className)}

// ✅ Data attribute state (từ React Aria):
"data-loading:bg-brand-solid_hover"  // khi data-loading={true}
"data-icon-only:p-2"                 // khi data-icon-only={true}
"group-invalid/input:block"          // khi parent group invalid

// ❌ KHÔNG hard-code màu:
className="bg-[#7f56d9]"  // sai

// ❌ KHÔNG dùng `text-base` (dùng `text-md`):
className="text-base"  // sai

// ❌ KHÔNG dùng "dark:" prefix:
className="dark:bg-gray-900"  // sai — dùng semantic token, dark mode tự handle
```

## 10. Checklist khi tạo component mới

Khi AI sinh code cho một component mới trong `components/base/`, phải đảm bảo:

- [ ] `"use client"` ở đầu file
- [ ] Import `cx`, `sortCx` từ `@/utils/cx`
- [ ] Import `isReactComponent` từ `@/utils/is-react-component` nếu nhận icon prop
- [ ] Wrap `<button>` bằng `AriaButton`, `<a>` bằng `AriaLink`, `<input>` bằng `AriaInput + AriaGroup`
- [ ] Prop boolean dùng `isDisabled`, `isLoading`, `isInvalid`, `isRequired` (không phải `disabled`, `loading`...)
- [ ] Style object khai báo với `sortCx({})` chia theo `common`, `sizes`, `colors`
- [ ] Không hard-code màu hex — chỉ dùng Tailwind class tham chiếu đến token
- [ ] Export type `Props` và component theo tên file (PascalCase)
- [ ] Nếu component có `tooltip` prop, tự wrap `<Tooltip>` bên trong
- [ ] Hỗ trợ `className` prop để override từ ngoài
- [ ] Tên file: kebab-case, tên component: PascalCase
